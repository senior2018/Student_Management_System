# Student Management System — System Documentation

Technical reference for developers.

For end-user instructions see [USER_MANUAL.md](USER_MANUAL.md).

---

## 1. Overview

A simple, single-admin web app for managing students. Built with plain PHP (no framework) and MySQL. Designed as a learning project for first-time PHP/MySQL students.

**Core modules:**
1. Login / authentication
2. Student CRUD (Create, Read, Update, Delete) + search
3. Attendance management
4. Grades + GPA calculation
5. Photo upload
6. PDF report generation
7. Email notifications

## 4. Database Schema

Database name: **`student`** (configurable in `includes/config.php`).

### Table: `users`
Admin accounts.

| Column | Type | Notes |
|---|---|---|
| id | INT, PK, AUTO_INCREMENT | |
| username | VARCHAR(50), UNIQUE | login name |
| password | VARCHAR(255) | bcrypt hash (NEVER plain text) |
| created_at | TIMESTAMP | auto-set |

### Table: `students`
Student records.

| Column | Type | Notes |
|---|---|---|
| id | INT, PK, AUTO_INCREMENT | |
| fullname | VARCHAR(100) | |
| gender | ENUM('Male','Female','Other') | |
| course | VARCHAR(100) | |
| email | VARCHAR(100), UNIQUE | |
| phone | VARCHAR(20) | |
| photo | VARCHAR(255), NULL | filename inside `/uploads/` |
| created_at | TIMESTAMP | |

### Table: `attendance`
One row per student per day.

| Column | Type | Notes |
|---|---|---|
| id | INT, PK, AUTO_INCREMENT | |
| student_id | INT, FK → students.id, ON DELETE CASCADE | |
| attend_date | DATE | |
| status | ENUM('Present','Absent','Late') | |
| created_at | TIMESTAMP | |

- **UNIQUE KEY** `(student_id, attend_date)` — prevents duplicates.
- INSERT uses `ON DUPLICATE KEY UPDATE` so re-marking the same day updates instead of failing.

### Table: `grades`
Subject scores per student for GPA calculation.

| Column | Type | Notes |
|---|---|---|
| id | INT, PK, AUTO_INCREMENT | |
| student_id | INT, FK → students.id, ON DELETE CASCADE | |
| subject | VARCHAR(100) | e.g. "Math 101" |
| score | DECIMAL(5,2) | 0–100 |
| credits | TINYINT UNSIGNED | weight (1–10) |
| created_at | TIMESTAMP | |

## 5. Installation

### Prerequisites
- macOS / Linux / Windows with:
  - PHP 8.0+ (`php -v`)
  - MySQL 5.7+ or MariaDB 10.3+ (`mysql --version`)
  - Composer 2.x (`composer --version`)
- A running MySQL server (`brew services start mysql` on macOS)

### Steps

```bash
# 1. Clone or copy the project
cd /path/to/Student_Management_System

# 2. Install PHP dependencies
composer install

# 3. Create the config file
cp includes/config.example.php includes/config.php
# Then edit includes/config.php and set the real MySQL password

# 4. Start the dev server (any free port)
php -S localhost:8080
```

Open in browser:
- **First time only**: <http://localhost:8080/setup.php> — creates the database, tables, and the default admin (`admin` / `admin123`).
- **DELETE setup.php** after it succeeds.
- Then go to <http://localhost:8080/login.php>.

## 6. Authentication & Sessions

Implemented in `includes/auth.php`.

- Passwords are hashed with `password_hash($p, PASSWORD_BCRYPT)`.
- Verification uses `password_verify()`.
- On login, `session_regenerate_id(true)` mints a new session ID (defends against session-fixation attacks).
- Session cookie is `httponly` and `samesite=Lax`.
- `require_login()` at the top of any protected page redirects unauthenticated visitors to `login.php`.

---

## 7. Security

| Threat | Mitigation |
|---|---|
| SQL injection | All queries use PDO prepared statements with named placeholders. No string concatenation in queries. |
| XSS (cross-site scripting) | All output that contains user data is escaped with `e()` which calls `htmlspecialchars($v, ENT_QUOTES, 'UTF-8')`. |
| CSRF (cross-site request forgery) | Every state-changing form includes a hidden `csrf_token` from `csrf_field()`. Handlers call `csrf_verify()` which uses `hash_equals()` (timing-safe comparison). |
| Password storage | bcrypt via `password_hash()` — never plain text. |
| File upload | MIME type checked with `finfo`, not the browser-provided type. Extension whitelist (jpg/png/gif/webp). Size limit 2 MB. Files are saved with a random unique name to prevent collisions. |
| Session fixation | `session_regenerate_id(true)` on login. |
| Direct script execution | `delete_student.php` rejects anything that isn't a POST request. `setup.php` is a one-off and meant to be deleted. |

## 8. Key Functions Reference

### `includes/db.php`
- **`db(): PDO`** — Lazy-loaded singleton PDO connection. Reads config, opens once, returns same object on subsequent calls.

### `includes/auth.php`
- `is_logged_in(): bool`
- `require_login(): void`
- `current_user(): ?array`
- `login_user(array $user): void` — call after credentials are verified
- `logout_user(): void`

### `includes/helpers.php`
- `e($v): string` — HTML-escape
- `csrf_token(): string`
- `csrf_field(): string` — emits hidden input
- `csrf_verify(): void` — kills the request on mismatch
- `flash_set($key, $msg)`, `flash_get($key)` — one-time messages
- `validate_student($input): [errors, cleaned]`
- `handle_photo_upload($file, $oldPhoto = null): ?string`
- `photo_url($filename): string`
- `score_to_gpa($score): float` — TCU 5.0 scale: 70+=5.0, 60+=4.0, 50+=3.0, 40+=2.0, 35+=1.0, else 0.0
- `score_to_letter($score): string` — A, B+, B, C, D, F
- `calculate_gpa($grades): ?float` — weighted by credits, returns null if no grades
- `gpa_classification($gpa): string` — "First Class Honours", "Upper Second Class Honours", etc.

### `includes/mail.php`
- `mail_config(): ?array` — returns `null` if email isn't set up
- `build_mailer(): ?PHPMailer`
- `send_welcome_email($email, $name): bool`

## 9. CRUD Flow Walkthrough (Students)

**Create** (`add_student.php`)
1. GET → show empty form
2. POST →
   1. `csrf_verify()`
   2. `validate_student($_POST)` returns errors + cleaned data
   3. If valid: `handle_photo_upload()` then `INSERT INTO students`
   4. Try `send_welcome_email()` (silent fail)
   5. `flash_set('success', ...)` and redirect to list

**Read** (`view_students.php`)
- GET → SELECT (with optional `LIKE`/`id` filter), compute each student's GPA, render table.

**Update** (`edit_student.php`)
1. GET with `?id=N` → load student → fill form
2. POST → validate → `handle_photo_upload(file, $student['photo'])` keeps the old photo if no new one → `UPDATE students` → redirect

**Delete** (`delete_student.php`)
- POST only. Reject GET. `csrf_verify()`. `DELETE FROM students WHERE id = :id`. Attendance + grades cascade-delete automatically.

## 10. Email Setup

By default, the `mail` section in `includes/config.php` contains placeholders (`YOUR_GMAIL_ADDRESS`, `YOUR_GMAIL_APP_PASSWORD`). When `mail_config()` sees the placeholder, it returns `null` and emails are silently skipped — the app works either way.

**To enable Gmail SMTP:**

1. Use a Google account that has 2-step verification turned on.
2. Generate an **App Password** at <https://myaccount.google.com/apppasswords>.
3. Edit `includes/config.php`:
   ```php
   'mail' => [
       'host'       => 'smtp.gmail.com',
       'port'       => 587,
       'encryption' => 'tls',
       'username'   => 'you@gmail.com',
       'password'   => 'xxxx xxxx xxxx xxxx',   // the 16-char app password
       'from_email' => 'you@gmail.com',
       'from_name'  => 'Student Management System',
   ],
   ```
4. Reload the dashboard — System Status should now say **Active**.

For other SMTP providers (SendGrid, Mailgun, etc.) change `host`, `port`, and `encryption` accordingly.

## 10b. GPA Calculation (TCU 5.0 Scale)

The system uses the **Tanzania Commission for Universities (TCU)** 5.0-point grading scale.

### Score → Grade Point conversion

| Score (%) | Letter | Grade Point | Meaning |
|---|---|---|---|
| 70 – 100 | A  | 5.0 | Excellent |
| 60 – 69  | B+ | 4.0 | Very Good |
| 50 – 59  | B  | 3.0 | Good (minimum pass) |
| 40 – 49  | C  | 2.0 | Satisfactory |
| 35 – 39  | D  | 1.0 | Marginal Fail |
| 0 – 34   | F  | 0.0 | Absolute Fail |

Implemented in `score_to_gpa($score)` and `score_to_letter($score)` in `includes/helpers.php`.

### GPA formula

For each course:

```
Quality Points (QP) = Grade Point × Credit Units
```

For the overall GPA:

```
GPA = ΣQP / ΣCredit Units
```

Implemented in `calculate_gpa($grades)`. Returns `null` when the student has no grades yet.

### Degree classification

`gpa_classification($gpa)` returns:

| GPA | Classification |
|---|---|
| 4.4 – 5.0 | First Class Honours |
| 3.5 – 4.3 | Upper Second Class Honours |
| 2.7 – 3.4 | Lower Second Class Honours |
| 2.0 – 2.6 | Pass |
| Below 2.0 | Fail |

Shown next to the GPA on `grades.php` and in the **Classification** column of the PDF report.

## 11. PDF Report

`report.php` builds an HTML string with the student list (including GPA) and passes it to dompdf:

```php
$dompdf->loadHtml($html);
$dompdf->setPaper('A4', 'landscape');
$dompdf->render();
$dompdf->stream('students-YYYY-MM-DD.pdf', ['Attachment' => false]);
```

`'Attachment' => false` means the PDF displays in the browser tab. Change to `true` to force download.

dompdf supports a subset of CSS — the inline `<style>` in `report.php` uses only that subset.

## 12. Responsive Design

`assets/style.css` uses three breakpoints:

| Width | Changes |
|---|---|
| ≤ 768px | Smaller paddings, smaller headings, 2-column stat grid |
| ≤ 640px | Stacked top bar, 1-column stats, full-width search controls, horizontally scrolling tables |
| ≤ 380px | Even smaller spacing for tiny phones |

Form inputs use `font-size: 16px` on phones to prevent iOS Safari from auto-zooming on focus.
